## ML Models

### 1. Heart Disease Predictor
- **Algorithm**: Ensemble (GradientBoosting + RandomForest + HistGradientBoosting)
- **Features**: 13 clinical features (age, BP, cholesterol, ECG, etc.)
- **Accuracy**: ~85-90%
- **Dataset**: UCI Cleveland Heart Disease (synthetic)

### 2. Diabetes Risk Classifier
- **Algorithm**: GradientBoosting + Feature Selection
- **Features**: 8 clinical features (glucose, BMI, insulin, age, etc.)
- **Accuracy**: ~78-85%
- **Dataset**: Pima Indians Diabetes (synthetic)

### 3. Breast Cancer Classifier
- **Algorithm**: SVM (RBF kernel) + PCA
- **Features**: 30 cell nucleus features
- **Accuracy**: ~97%
- **Dataset**: Wisconsin Breast Cancer (real data from sklearn)

## Upgrade Path

### When to Upgrade

**From SQLite to PostgreSQL**:
- When you have >10,000 patients
- When you need multi-instance scaling
- When you need better concurrency

**From Free Tier to Paid**:
- Backend: No cold starts, more memory, faster CPUs
- Frontend: Unlimited bandwidth, custom analytics

### How to Upgrade

1. **Add PostgreSQL**:
   - Render: Dashboard → Add PostgreSQL database
   - Update `DATABASE_URL` env var
   - Run migrations: `alembic upgrade head`

2. **Upgrade Render Plan**:
   - Dashboard → Settings → Change Plan
   - Increase workers to 4
   - Add more memory

3. **Add Monitoring**:
   - Integrate Sentry for error tracking
   - Add LogDNA/Papertrail for logs
   - Set up Uptime Robot for availability
